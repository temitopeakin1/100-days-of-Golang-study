<strong>100 days of Golang<Strong>

✅ Foundational Setup (Day 1–7) – "Getting Started with Go"
📌 Goals:
* Understand what Go is, why it was created, and what makes it unique.
* Set up Go environment and tools.
🔹 Daily Tasks:
1. Install Go & Setup Workspace – https://golang.org/doc/install
2. Learn Go CLI: go run, go build, go mod
3. Your first “Hello, World!”
4. Understand package main, func main()
5. Data types & variables
6. Constants, Operators
7. Basic I/O with fmt
Resources:
* Tour of Go
* Go official docs: https://golang.org/doc/

✅ Week 2 (Days 8–14) – "Core Programming Constructs"
📌 Goals:
* Master control structures and functions
🔹 Topics:
* If/else, switch
* Loops (for, range)
* Functions (multiple return values, named returns)
* Error handling with error type
* Pointers
* Packages and Imports
📚 Practice:
* Mini projects like a simple calculator, number guess game, etc.

✅ Week 3 (Days 15–21) – "Arrays, Slices, Maps, and Structs"
📌 Goals:
* Understand Go’s data structures
🔹 Topics:
* Arrays vs Slices
* Map (Go's built-in hash table)
* Structs and field access
* Nesting structs
* JSON marshaling/unmarshaling
Practice:
* Build a contact book CLI
* JSON-based data store

✅ Week 4 (Days 22–28) – "Concurrency in Go"
📌 Goals:
* Understand why Go is famous for concurrency
🔹 Topics:
* Goroutines
* Channels
* Buffered vs Unbuffered Channels
* select statement
* Mutexes, WaitGroups
* Real-world: build a concurrent web crawler

✅ Week 5 (Days 29–35) – "Files, Error Handling, and Logging"
📌 Goals:
* Read/write files, advanced error management
🔹 Topics:
* os, io/ioutil, bufio, encoding/csv
* Logging with log package
* Error wrapping & custom errors
* Defer, Panic, Recover

✅ Week 6 (Days 36–42) – "Testing and Benchmarking"
📌 Goals:
* Learn Go’s built-in testing tools
🔹 Topics:
* testing package
* Unit tests
* Table-driven tests
* Mocks
* Benchmarking functions

✅ Week 7–8 (Days 43–56) – "Object-Oriented Go & Interfaces"
📌 Goals:
* Master Go’s way of abstraction
🔹 Topics:
* Interfaces and type assertions
* Composition over inheritance
* Polymorphism
* Dependency injection
Practice:
* Build a payment processing system using interfaces

✅ Week 9 (Days 57–63) – "Go Web Development"
📌 Goals:
* Build RESTful APIs
🔹 Topics:
* net/http
* HTTP Handlers & Routers
* Use third-party router like gorilla/mux or chi
* Serve JSON, handle HTTP status codes
* Middleware pattern
Practice:
* Create a simple blog or ToDo REST API

✅ Week 10 (Days 64–70) – "Databases with Go"
📌 Goals:
* Connect to databases, handle data persistence
🔹 Topics:
* SQL with Go (database/sql)
* PostgreSQL or SQLite
* ORM libraries: GORM, sqlx
* Migrations with goose or golang-migrate
Practice:
* Expand blog API to store posts in a DB

✅ Week 11 (Days 71–77) – "Advanced Topics"
📌 Topics:
* Reflection
* Generics (Go 1.18+)
* Context package (managing cancellation, deadlines)
* Interfaces in depth
* Go Modules (go mod tidy, versioning, replacing)

✅ Week 12–13 (Days 78–91) – "Microservices, APIs, and Production Tools"
📌 Goals:
* Deploy and structure production-ready Go services
🔹 Topics:
* RESTful vs gRPC (use protobuf)
* JWT Authentication
* Environment variables (godotenv)
* Logging with zap or logrus
* Dockerize your Go app
* CI/CD basics with GitHub Actions

✅ Week 14 (Days 92–100) – "Capstone Projects and Deployment"
📌 Goals:
* Consolidate skills into real-world projects
Ideas:
* Blog API with DB, JWT auth, and Docker
* CLI tool (e.g., a task manager)
* Chat app using WebSockets
* Personal finance tracker
Deployment:
* Deploy on Render, Railway, Vercel backend, or DigitalOcean
* Set up monitoring/logging

🔧 Tools You Should Learn Along the Way:
* GoLint, GoFmt, GoVet
* Delve for debugging
* Go Modules
* Swagger/OpenAPI for API docs
* VSCode + Go Plugin
